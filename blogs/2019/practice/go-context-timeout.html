    Context超时实现

    Hyman 2019/9/13 实践总结Go

    go语言中的context常用作实现超时功能，一般做法是在请求接收时创建一个有超时时间的context，然后将这个context作为第一个参数传入接下来的的调用方法中，是不是这样就好了？！当然不是，接下来在就要在需要超时限制的的地方开一个协程来执行可能超时的逻辑（例如调用第三方接口），主进程使用select来等待超时或请求结束，例如：

    package main
 
import (
	"context"
	"fmt"
	"time"
)
 
func main() {
	// after 2 second time out
	ctx, cancel :=context.WithTimeout(context.Background(), 5*time.Second)
	defer cancel()
	handle(ctx)
}
func handle(ctx context.Context) {
	resp:=make(chan error, 1)
	go func() {
		defer close(resp)
		resp<- run(ctx)
	}()
	select {
	case <-ctx.Done():
        <-resp // wait for run return
		fmt.Println("handle time out")
	case <-resp:
		fmt.Println("response")
	}
}
// business logic
func run(ctx context.Context) error {
	time.Sleep(10*time.Second)
	return nil
}

    handle方法中超时后，run逻辑并不会马上停止，实际上还是会继续执行，直到运行结束，如果一个调用链很长， 超时后，这个调用链还是会继续运行占用资源，所以在实际开发中，如果run还有调用其他接口，会把当前的context继续作为第一个参数继续传入接下来的方法中，重复handler的逻辑，这样就可以保证资源能最大程度释放。
